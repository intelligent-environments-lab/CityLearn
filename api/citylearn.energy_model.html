autosize(demand: float, duration: float | Tuple[float, float] = None, parallel: bool = None, safety_factor: float | Tuple[float, float] = None, sizing_data: DataFrame = None) → Tuple[float, float, float, float, float, float][source]ÔÉÅ

Randomly selects a battery from the internally defined real world manufacturer model and autosizes its parameters.

The total capacity and nominal power are autosized to meet the hourly demand for a specified duration. It is assumed that there is no limit on the number of batteries that can be connected in series or parallel for any of the battery models.

Parameters:

  • demand (float) ‚Äì Hourly, building demand to be met for duration.

  • duration (Union[float, Tuple[float, float]], default : (1.5, 3.5)) ‚Äì Number of hours the sized battery should be able to meet demand.

  • parallel (bool, default : False) ‚Äì Whether to assume multiple batteries are connected in parallel so that the maximum nominal power is the product of the unit count and the nominal_power of one battery i.e., increasing number of battery units also increases nominal power.

  • safety_factor (Union[float, Tuple[float, float]], default: 1.0) ‚Äì The target capacity is oversized by factor of `safety_factor.

Returns:

  • capacity (float) ‚Äì Selected battery‚Äôs autosized capacity to meet demand for duration.

  • nominal_power (float) ‚Äì Selected battery‚Äôs autosized nominal power to meet demand for duration.

  • depth_of_discharge (float) ‚Äì Selected battery depth-of-discharge.

  • efficiency (float) ‚Äì Selected battery efficiency.

  • loss_coefficient (float) ‚Äì Selected battery loss coefficient.

  • capacity_loss_coefficient (float) ‚Äì Selected battery capacity loss coefficient.

  • sizing_data (pd.DataFrame, optional) ‚Äì The sizing dataframe from which batteries systems are sampled from. If initialized from py:class:citylearn.citylearn.CityLearnEnv, the data is parsed in when autosizing a building‚Äôs battery. If the dataframe is not provided it is read in using citylearn.data.DataSet.get_battery_sizing_data().

Notes

Data source: https://github.com/intelligent-environments-lab/CityLearn/tree/master/citylearn/data/misc/battery_choices.yaml.

Autosize nominal_power and inverter_ac_power_per_kw.

Samples PV data from Tracking the Sun dataset to set PV system design parameters in System Adivosry Model‚Äôs PVWattsNone model. The PV is sized to generate zero_net_energy_proportion of annual_demand limited by the roof_area. It is assumed that the building‚Äôs roof is suitable for the installation tilt and azimuth in the sampled data.

Parameters:

  • demand (float) ‚Äì Building annual demand in [kWh].

  • epw_filepath (Union[Path, str]) ‚Äì EnergyPlus weather file path used as input to PVWattsNone model.

  • use_sample_target (bool, default: False) ‚Äì Whether to directly use the sizing in the sampled instance instead of sizing for zero_net_energy_proportion. Will still limit the size to the roof_area.

  • zero_net_energy_proportion (Union[float, Tuple[float, float]], default: (0.7, 1.0)) ‚Äì Proportion

  • roof_area (float, optional) ‚Äì Roof area where the PV is mounted in m^2. The default is to assume an infinite roof area.

  • safety_factor (Union[float, Tuple[float, float]], default: 1.0) ‚Äì The nominal_power is oversized by factor of safety_factor. It is only applied to the zero_net_energy_proportion estimate.

  • sizing_data (pd.DataFrame, optional) ‚Äì The sizing dataframe from which PV systems are sampled from. If initialized from py:class:citylearn.citylearn.CityLearnEnv, the data is parsed in when autosizing a building‚Äôs PV. If the dataframe is not provided it is read in using citylearn.data.DataSet.get_pv_sizing_data().

Returns:

  • nominal_power (float) ‚Äì Autosized nominal power.

  • inverter_ac_power_per_kw (np.ndarray) ‚Äì SAM ac output for PVWattsNone model.

Notes

Data source: https://github.com/intelligent-environments-lab/CityLearn/tree/master/citylearn/data/misc/lbl-tracking_the_sun_res-pv.csv.
